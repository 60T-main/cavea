# ინვენტარის მართვა

## ტექნოლოგიები

- Frontend: React (TypeScript), Bootstrap 5
- Backend: Node.js 22+, ExpressJS, Sequelize v6, TypeScript
- მონაცემთა ბაზა: PostgreSQL

## პროექტის სტრუქტურა

```
cavea/
  frontend/
    ...
  backend/
    ...
```

## ინსტრუქცია გაშვებისათვის

### 1. რეპოზიტორიის დაკლონვა

```
git clone https://github.com/60T-main/cavea
cd cavea
```

### 2. მონაცემთა ბაზის კონფიგურაცია

- შექმენით PostgreSQL ბაზა და მომხმარებელი.
- შექმენით `.env` backend-ში და შეავსეთ თქვენი მონაცემები:

```
DB_NAME=თქვენი_ბაზის_სახელი
DB_USER=თქვენი_მომხმარებელი
DB_PASS=თქვენი_პაროლი
DB_HOST=localhost
```

### 3. დამოკიდებულებები

#### Backend

```
cd backend
npm install
```

#### Frontend

```
cd ../frontend
npm install
```

### 4. ბაზის ინიციალიზაცია და სატესტო მონაცემების გენერაცია

- 500,000 ინვენტარის ჩანაწერის გენერაცია:

```
npx ts-node src/generateInventories.ts
```

### 5. Backend-ის გაშვება

```
cd backend
npm start
```

### 6. Frontend-ის გაშვება

```
cd frontend
npm run dev
```

## ფუნქციონალი

- ინვენტარის ცხრილი (20 ჩანაწერი თითო გვერდზე)
- ინვენტარის დამატება (ფორმა ვალიდაციით)
- ინვენტარის წაშლა
- ადგილმდებარეობის მიხედვით ფილტრი (ფილტრაცია ბექში)
- სორტირება: სახელით, ფასით, ადგილმდებარეობით (ბექში)
- სტატისტიკის მოდალი (კინოთეატრის მიხედვით რაოდენობა და ჯამური ფასი)

## API Endpoint-ები

### ინვენტარი

- `GET /inventories` — ინვენტარის სია (პაგინაცია, ფილტრი, სორტირება)
- `POST /inventories/add` — ინვენტარის დამატება
- `DELETE /inventories/:id` — ინვენტარის წაშლა

### ლოკაცია

- `GET /locations` — ლოკაციების სია
- `POST /locations/add` — ლოკაციის დამატება
- `GET /locations/statistics` — სტატისტიკა ლოკაციის მიხედვით

## დიდი მონაცემების ტესტირება

- გამოიყენეთ სკრიპტები 500,000 ჩანაწერის გენერაციისთვის.
- შეგიძლიათ რამდენჯერმე გაუშვათ ეს სკრიპტები, ისინი ავტომატურად გაასუფთავებენ და ხელახლა შეავსებენ ბაზას.
